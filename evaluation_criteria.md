# JLPT 일본어 문법 교재 검수 기준 (evaluation_criteria.md)

본 문서는 N5~N1 일본어 문법 교재(.md 11개 파일)의 검수 기준을 정의한다.
검수자는 본 기준에 따라 각 문법 항목을 평가하고, CLAUDE.md에 정의된 표 형식 보고서로 이슈를 기록한다.

---

## 1. 검수 단위와 범위

| 단위 | 범위 |
|------|------|
| 항목 | 「### ～문형」 단위 한 개 (한 챕터당 다수) |
| 한 세션 처리량 | 최대 10개 항목 (초과 시 전반·후반 분할) |
| 검수 대상 | 의미·기능, 접속 형태, 예문, ❌/⭕️ 쌍, 비교, 한국어 화자 주의, 장면 대화, 미니 활용표, 챕터 간 교차 참조 |

---

## 2. 이슈 유형 분류 (보고서 「유형」 칸 표기)

| 유형 코드 | 분류 | 적용 대상 |
|-----------|------|-----------|
| 文法 | 일본어 문법 정확성 | 접속 형태·활용·용법 설명의 오류 |
| 例文 | 예문 자연성·정확성 | 일본어 예문 자체의 오류·부자연스러움 |
| 訳 | 한국어 번역 | 번역의 정확성·자연성·뉘앙스 |
| 対比 | ❌/⭕️ 대비 일관성 | 두 예문 간 의미·구조의 논리적 정합성 |
| ruby | 후리가나 태그 | 누락·오류·표기 불일치 |
| 登録 | 레지스터 태그 | 🔄/📝/🎩/👥 표기 일관성·적절성 |
| 表 | 비교·활용 표 내용 | 표 내 정보의 정확성·완전성 |
| 場面 | 장면 대화문 | 회화의 자연성·역할 적절성 |
| 参照 | 교차 참조 | 챕터·항목 간 표기·연결 |
| 体裁 | 형식·체재 | 마크다운·기호·들여쓰기·번호 |

## 3. 항목별 세부 검수 기준

### 3.1 文法 — 일본어 문법 설명의 정확성

**접속 형태 (■ 접속 형태)**
- [ ] 동사 그룹별(1·2·3그룹) 접속 규칙이 정확한가
- [ ] い형용사·な형용사·명사 접속 규칙이 분리되어 정확히 기술되었는가
- [ ] 보통형/정중형, 과거/비과거, 긍정/부정의 활용 표시가 정확한가
- [ ] 「⚠️」 경고로 직접 접속 불가 품사를 명시했는가 (예: い형용사에 だ 직접 접속 불가)
- [ ] 음편(イ音便·促音便·撥音便) 또는 불규칙 활용을 빠뜨리지 않았는가
- [ ] 활용 패러다임 표의 활용형이 표준 일본어 사전과 일치하는가

**용법 설명 (■ 의미와 기능)**
- [ ] 핵심 의미가 사전적 정의와 일치하는가
- [ ] 복수 용법이 있는 경우 모두 분리·번호화하여 제시했는가 (예: 직접 수동 / 간접 수동, 강제 / 허용)
- [ ] 뉘앙스(불만·피해·격식 등)가 정확히 기술되었는가
- [ ] 함께 쓰이는 조사·부사의 호응 패턴이 정확한가

### 3.2 例文 — 예문과 설명의 자연성

- [ ] 일본어 예문이 자연스러우며 모어 화자가 실제로 사용하는 표현인가
- [ ] 활용형이 표제 문형의 접속 규칙과 일치하는가
- [ ] 어휘 난이도가 해당 JLPT 레벨에 부합하는가
  - N5: 초급 어휘, 기본 한자
  - N4: 일상 회화 어휘
  - N3~N1: 추상·문어·전문 어휘 단계적 확대
- [ ] 다의 어휘 사용 시 문맥상 의도한 의미로 해석 가능한가
- [ ] 한자 사용 빈도와 후리가나 부착이 레벨에 맞는가
- [ ] 예문 사이의 다양성(문체·주어·시제·긍정·부정)이 충분한가

### 3.3 訳 — 한국어 번역의 정확성과 자연성

- [ ] 일본어 원문의 의미를 누락·왜곡 없이 전달하는가
- [ ] 한국어로 자연스러운 어순·조사·종결어미를 사용하는가
- [ ] 일본어 특유 표현(간접 수동, 수수 표현, 경어 등)을 한국어 화자가 이해 가능한 의역으로 처리했는가
- [ ] 직역과 의역이 충돌할 때 「(직역: ~)」 등으로 보충했는가
- [ ] 일본어 존경·겸양·정중의 단계가 한국어 존댓말 단계와 적절히 대응되는가
- [ ] 화자·청자의 인칭 대명사가 한국어 관습상 자연스러운가 (불필요한 「나는」 등의 반복 회피)
- [ ] 「(의도한 의미)」 표기가 ❌ 예문에 일관되게 부착되었는가

### 3.4 対比 — ❌/⭕️ 예문 쌍의 논리적 일관성

- [ ] ❌ 예문과 ⭕️ 예문이 동일한 의미를 의도하고 있는가 (수정 전후의 의미 동일성)
- [ ] ⭕️로 제시된 수정안이 실제로 ❌의 문제를 해결하는가
- [ ] ※(주석)으로 ❌가 왜 오류인지 명확히 설명되었는가
- [ ] ❌ 예문이 학습자가 실제로 범하는 오류를 반영하는가 (인공적·비현실적 오류 회피)
- [ ] ⭕️ 예문이 단순한 회피가 아니라 표제 문형을 사용한 정답인가
- [ ] 한 문형 내에서 다중의 ❌/⭕️ 쌍이 있을 때 각 쌍이 서로 다른 오류 유형을 다루는가

### 3.5 ruby — 후리가나 태그 정확성

- [ ] 한자 노출 시 `<ruby>漢字<rt>かな</rt></ruby>` 형식이 정확한가
- [ ] 한자 1자에 대응하는 가나가 정확하게 분리되었는가
- [ ] 같은 한자가 같은 파일 내에서 일관된 후리가나로 표기되었는가 (「<ruby>明日<rt>あした</rt></ruby>」 vs 「<ruby>明日<rt>あす</rt></ruby>」 등 의도적 차이는 허용)
- [ ] 닫는 태그 `</ruby>`, `</rt>`의 누락·중첩이 없는가
- [ ] 음독·훈독 혼동(예: 「<ruby>大切<rt>たいせつ</rt></ruby>」를 「だいせつ」로 표기) 오류가 없는가
- [ ] 레벨에 맞는 한자에만 ruby가 부착되었는가 (N5에서는 모든 한자, N1에서는 고급 한자만 등)
- [ ] 본문·표·예문의 ruby 부착 정책이 한 파일 내에서 일관되는가

### 3.6 登録 — 레지스터 태그(🔄/📝/🎩/👥) 일관성

- [ ] 표제어 옆에 「[ 아이콘 설명 ]」 형식의 레지스터 태그가 부착되어 있는가
- [ ] 사용된 아이콘이 다음 정의와 일치하는가
  - 🔄 구어·문어 모두
  - 📝 문어·격식
  - 🎩 격식·정중·존경 (경어 계열)
  - 👥 구어·일상 (회화 한정)
- [ ] 동일 문형이 여러 챕터에서 등장할 때 레지스터 태그가 일관되는가
- [ ] 비교 표의 「레지스터」 행 표기가 본문 표제어 태그와 일치하는가
- [ ] 「구어에서는 ～」 「문어에서는 ～」 등의 본문 설명과 태그가 모순되지 않는가

### 3.7 表 — 비교·활용·일람 표의 정확성

**비교 표 (■ 비교)**
- [ ] 비교 대상이 학습자가 혼동하기 쉬운 문형인가
- [ ] 비교 항목(기능·뉘앙스·접속·예·레지스터)이 일관된 기준으로 작성되었는가
- [ ] 표 내 정보가 각 문형의 본문 설명과 모순되지 않는가
- [ ] 예문 칸의 일본어가 정확하고 단순한가

**활용 패러다임 표**
- [ ] 1·2·3그룹 동사가 망라되어 있는가
- [ ] 불규칙 동사(する·くる·ある·行く 등)가 별도 행으로 처리되었는가
- [ ] 활용형이 사전 표제어 표기와 일치하는가

**일람표 (예: こそあど 체계, 의문사, 조수사)**
- [ ] 체계의 빈칸이 누락되지 않았는가
- [ ] 가로·세로축의 분류 기준이 명확한가

### 3.8 場面 — 장면 대화문의 자연성

- [ ] 등장인물의 발화가 그 역할(상사·부하·점원·손님·친구·가족 등)에 어울리는가
- [ ] 경어·반말의 사용이 인물 관계와 일치하는가
- [ ] 화제·장면(직장·식당·학교 등)이 표제 문형의 전형적 사용 맥락을 반영하는가
- [ ] 발화의 길이·턴테이킹이 실제 회화답게 자연스러운가
- [ ] 종조사(ね·よ·わ·ぞ 등)의 사용이 화자 성별·관계에 맞는가
- [ ] 한국어 번역에서 화자별 말투(존댓말·반말)가 일본어와 일치하는가

### 3.9 参照 — 챕터 간 교차 참조의 정확성

- [ ] 「→ Ch.X 참조」 「N4에서 배운 ～」 등의 참조 표기가 실제 위치를 정확히 가리키는가
- [ ] 참조 대상 문형명이 해당 챕터의 표제어와 정확히 일치하는가
- [ ] 레벨 간 참조(N5 → N4 등)가 학습 순서상 모순되지 않는가
- [ ] 「→ ～だ / です의 비교 참조」 같은 짧은 참조가 동일 챕터 내 실재하는 비교 표를 가리키는가
- [ ] 학습 안내(💡 학습 안내)에서 제안한 학습 순서가 목차와 일치하는가

### 3.10 体裁 — 형식·체재

- [ ] 챕터·문형 표제의 마크다운 헤더 레벨이 일관되는가 (`##` `###` `####` 등)
- [ ] 「■」 「💡」 「⚠️」 「📋」 「❌」 「⭕️」 등 기호의 의미·사용이 일관되는가
- [ ] 예문 번호가 전각 숫자(１. ２.)로 일관되게 표기되었는가
- [ ] 예문 줄바꿈 후 화살표 「　→」 들여쓰기(전각 공백)가 일관되는가
- [ ] 「(～)」 안 보충 설명의 괄호 종류(반각·전각)가 일관되는가
- [ ] 표의 헤더 행 구분자 「|---|---|」 누락이 없는가
- [ ] 표제 옆 한국어 의미 「（~~~）」의 양식이 일관되는가

## 4. 보고서 작성 규칙 (CLAUDE.md 준수)

### 4.1 형식 (서술형 금지)

```
### [파일명] Ch.X [챕터 제목] (항목 N~M)
| 항목 | 줄번호 | 유형 | 문제 내용 | 수정안 |
|------|--------|------|-----------|--------|
| ～について | 196 | 訳 | "이야기했다"보다 "발표했다"가 자연 | "발표했다"로 변경 |
```

이슈가 없으면 「**이슈 없음**」 한 줄만 기재한다.

### 4.2 「항목」 칸
- 검수 대상 문형의 표제어를 그대로 인용 (예: `～について`, `～だ / です`)

### 4.3 「줄번호」 칸
- 마크다운 파일 기준 시작 줄번호 (Read 도구의 line number 기준)
- 범위 표기 가능: `196-201`

### 4.4 「유형」 칸
- §2의 코드 사용 (文法·例文·訳·対比·ruby·登録·表·場面·参照·体裁)
- 복합 이슈는 슬래시로 연결: `例文/訳`

### 4.5 「문제 내용」 칸
- 한 문장으로 핵심만 기술
- "~가 ~하다" 형식의 단정적 진술

### 4.6 「수정안」 칸
- 구체적 대체 표현·삭제·추가 위치 제시
- 단순 지적이 아닌 실행 가능한 수정안

## 5. 우선순위 (이슈가 많을 때)

1. **즉시 수정 필요 (Critical)**: 文法·例文 — 학습자에게 잘못된 문법을 가르치는 오류
2. **수정 권장 (Major)**: 訳·対比·表 — 의미 이해를 방해하는 오류
3. **개선 사항 (Minor)**: ruby·登録·参照·体裁 — 일관성·완성도 문제
4. **선택 개선 (Suggestion)**: 場面 — 자연스러움 향상

보고서에서 우선순위 표기는 생략한다. 「유형」 칸으로 추정 가능하다.

## 6. 검수자 행동 원칙

1. **추측 금지**: 확신 없는 일본어 문법 판단은 보고하지 않는다 (오탐지 방지)
2. **레벨 존중**: N5 교재의 단순화된 설명을 N1 기준으로 판단하지 않는다
3. **저자의 의도 우선**: 명백한 오류가 아니면 저자의 표현 선택을 존중한다
4. **출력 최소화**: 이슈 없는 항목은 「이슈 없음」으로만 기재, 불필요한 칭찬·총평 금지
5. **append만 수행**: 보고서 파일을 새로 쓰지 않고 기존 파일 끝에만 추가
