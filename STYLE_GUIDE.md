# STYLE_GUIDE.md
# JLPT 문법 교재 시리즈 (N5–N1) 공통 스타일 가이드

> **이 문서의 목적**: N5–N1 전 레벨 교재 파일에 공통으로 적용되는 형식·내용·편집 규칙을 정의한다.  
> **적용 대상**: 교재 마크다운 파일(.md), JSON 데이터셋 파일, 집필 메모 파일  
> **AI 편집 도구 사용 시**: 이 문서를 먼저 읽고, 이 규칙이 모든 판단의 기준이 된다. 추가 지시가 없는 한 이 규칙을 따른다.

## 1. 루비 후리가나 형식 규칙

### 1.1 기본 형식

모든 한자에는 루비 후리가나를 붙인다. 형식은 다음과 같다:
```html
<ruby>漢字<rt>よみかた</rt></ruby>
```

### 1.2 태그 쌍 규칙

- 여는 태그 `<ruby>`와 닫는 태그 `</ruby>`는 반드시 쌍으로 존재해야 한다.
- `<rt>`와 `</rt>`도 반드시 쌍으로 존재해야 한다.
- 다음은 오류 패턴이다:
```html
<!-- 오류: <ruby>가 없음 -->
漢字</rt></ruby>

<!-- 오류: <rt>가 없음 -->
<ruby>漢字</ruby>

<!-- 오류: 닫는 태그 누락 -->
<ruby>漢字<rt>かんじ</ruby>
```

### 1.3 적용 대상 제한

| 적용 | 미적용 |
|------|--------|
| 한자（漢字） | 히라가나（ひらがな） |
| 한자 포함 복합어 | 가타카나（カタカナ） |
| | 로마자 |

- 히라가나 단어（例: `のんびり`, `ベル`）에 루비 태그를 붙이지 않는다.
- 가타카나 단어（例: `コーヒー`, `プロジェクト`）에 루비 태그를 붙이지 않는다.

### 1.4 복합어 처리

한자와 히라가나가 혼재한 단어는 한자 부분에만 루비를 붙인다:
```html
<!-- 올바름 -->
<ruby>読<rt>よ</rt></ruby>みます
<ruby>食<rt>た</rt></ruby>べる

<!-- 오류: 히라가나 부분에 루비 포함 -->
<ruby>読みます<rt>よみます</rt></ruby>
```

## 2. ⭕️❌ 예문 작성 규칙

### 2.1 ❌/⭕️ 사용 조건

`❌`와 `⭕️`는 **문장 자체의 문법 오류**에만 사용한다.

**문법 오류의 정의**: 해당 문법 항목의 접속 규칙, 품사 제약, 필수 공기 제약을 위반한 경우.

| 오류 유형 | 처리 기호 | 예시 |
|---------|---------|------|
| 접속 형태 오류（품사 제약 위반） | ❌/⭕️ | 형용사에 접속 불가한 표현에 형용사를 붙인 경우 |
| 필수 공기 제약 위반 | ❌/⭕️ | 부정 호응 필수 표현에 긍정형을 쓴 경우 |
| 불규칙 활용 오류（する → せ～가 아닌 경우） | ❌/⭕️ | `するざるを得ない` vs `せざるを得ない` |
| 뒤 절 제약 위반（의지 표현 금지 위치에 사용） | ❌/⭕️ | `が早いか〜しよう` |

### 2.2 ❌/⭕️ 사용 금지 조건

다음 경우에는 `❌/⭕️`를 사용하지 않는다. 대신 ⚠️ 또는 💡를 사용한다:

| 상황 | 올바른 처리 |
|------|----------|
| 해석 혼동（문법적으로는 가능하나 의미가 달라지는 경우） | ⚠️ 주의 또는 💡 참고 텍스트 |
| 자연스러움 차이（비문은 아니나 어색한 경우） | ⚠️ 주의 또는 💡 참고 텍스트 |
| 레지스터 부적합（격식 맥락에 구어 표현, 또는 반대） | ⚠️ 주의 텍스트 |
| 번역투（한국어 직역이 어색한 경우） | 💡 참고 텍스트 |

### 2.3 ❌ 예문 형식
```markdown
❌ 오류 문장
　→ 번역
　※ 오류 이유를 한 줄로 명시

⭕️ 올바른 문장
　→ 번역
```

- `※` 줄은 어떤 규칙을 위반했는지 1문장으로 명시한다.
- ❌ 예문은 항목 내 예문 목록의 **마지막에** 배치한다.

### 2.4 N1 특수 기준

N1 레벨에서 레지스터 부적합은 오류의 핵심 유형이다.  
그러나 레지스터 부적합에는 `❌/⭕️`를 쓰지 않고 `⚠️ 주의`로 처리한다.  
N1에서 `❌/⭕️`는 접속 형태 오류（불규칙 포함）와 필수 공기 제약 위반에만 사용한다.

## 3. 한국어 설명 문체 및 어조 규칙

### 3.1 언어 사용 원칙

**모든 설명, 주석, 장면 설명은 한국어로 작성한다.**

다음은 허용되는 일본어 사용 범위이다:

| 허용 | 불허 |
|------|------|
| 예문（일본어 문장） | 한국어 설명 문장 중간에 삽입된 일본어 |
| 접속 예시 표 내의 일본어 | 장면 설명란의 일본어 |
| 비교박스 표의 일본어 항목명 | 체크리스트의 일본어 |
| 항목 제목의 일본어 표기 | 주석·메모란의 일본어 문장 |

**금지 예시**:
```markdown
<!-- 금지: 한국어 문장 안에 일본어 삽입 -->
💡 2그룹 동사의 수동형「られる」는 가능형과 형태가 동일하다. 文脈で区別する.

<!-- 올바름 -->
💡 2그룹 동사의 수동형 「られる」는 가능형과 형태가 동일하다. 문맥으로 구별한다.
```

### 3.2 장면 설명 형식

장면 연속 예문 앞의 장면 설명은 반드시 한국어로 작성한다:
```markdown
<!-- 올바름 -->
［장면 설명: 직장 동료 두 명이 점심시간에 이야기하는 상황］

<!-- 금지 -->
［友だち同士でバーベキューの計画を立てている場面。］
```

### 3.3 체크리스트 언어 규칙

모든 `✅ 확인 체크리스트` 항목은 **한국어만** 사용한다.  
일본어 표현, 일본어 항목명이 체크리스트 내에 들어가서는 안 된다.
```markdown
<!-- 금지 -->
- [ ] ～なり와 ～が早いかの 차이를 설명할 수 있다

<!-- 올바름 -->
- [ ] 「～なり」와 「～が早いか」의 차이를 설명할 수 있다
```

단, 일본어 문법 표현 자체를 언급할 때는 「 」 안에 일본어를 사용할 수 있다.

### 3.4 금지 표현

다음 표현은 설명 내에 사용하지 않는다:

| 금지 표현 | 이유 |
|---------|------|
| "자연스럽게" | 기준이 불명확 |
| "적절히" | 기준이 불명확 |
| "보통" | 기준이 불명확 |
| "~하면 좋다" | 규칙인지 권장인지 불분명 |

대신 다음처럼 구체적으로 서술한다:

| 금지 | 올바른 표현 |
|------|----------|
| "자연스럽게 선택" | "챕터 내 동일 소재가 3회 이상 반복되지 않도록 선택" |
| "적절한 예문" | "해당 레지스터의 전형적 장면（뉴스·논설·소설·공문서 등）에서 선택" |

## 4. 레지스터 아이콘 규칙

### 4.1 아이콘 정의

| 아이콘 | 레지스터 | 적용 장면 |
|------|--------|---------|
| 💬 | 구어 | 친한 관계의 일상 대화, 반말 맥락 |
| 📝 | 문어·격식 | 뉴스, 논설, 보고서, 공문서, 비즈니스 문서 |
| 📖 | 문어·문학 | 소설, 수필, 고어, 문학적 서술 |
| 🔄 | 구어·문어 모두 | 격식 수준에 관계없이 사용 가능 |
| 🎩 | 경어·정중 | 공식적 경어 표현, 비즈니스 경어 |

### 4.2 아이콘 배치 위치

레지스터 아이콘은 **항목 제목 줄 끝**에 배치한다:
```markdown
### ～表現（~한국어 의미） ［📝 문어·격식］
```

### 4.3 레지스터 일관성 규칙

장면 연속 예문 내에서 **동일 대화의 같은 인물**은 레지스터를 일관되게 유지한다.

| 관계 유형 | 사용 가능 접속사 |
|---------|--------------|
| 친한 관계（친구, 가족 등 편한 사이） | けど, でも, だから, それで |
| 친한 관계에서 **금지** | しかし, だが, それゆえ, したがって, それに対して |
| 격식 있는 관계（직장 상하관계, 공식 발표 등） | しかし, したがって, そのため 모두 가능 |

## 5. 레벨 간 스테어케이스 정책

### 5.1 기본 원칙

각 레벨의 교재는 해당 레벨 이전에 학습된 문법·어휘만 예문과 설명에 사용한다.  
이를 **i+1 원칙**이라 한다: 각 항목은 새 문법 요소를 정확히 하나만 추가한다.

### 5.2 레벨별 전제 학습 범위

| 집필 레벨 | 예문·설명에서 전제할 수 있는 범위 |
|---------|--------------------------|
| N5 | N5 교재 해당 항목 이전의 N5 항목만 |
| N4 | N5 전체 + N4 해당 항목 이전 항목 |
| N3 | N5·N4 전체 + N3 해당 항목 이전 항목 |
| N2 | N5·N4·N3 전체 + N2 해당 항목 이전 항목 |
| N1 | N5·N4·N3·N2 전체 + N1 해당 항목 이전 항목 |

### 5.3 교차 참조 규칙

- 상위 레벨에서 하위 레벨 항목을 교차 참조할 수 있다: `"N4의 ～なければならない（E116）에서 심화"`
- 하위 레벨에서 상위 레벨 항목을 교차 참조하지 않는다.
- 동일 레벨 내에서 아직 등장하지 않은 항목을 예문에 사용하지 않는다.

### 5.4 나선형 연결 서술 형식

이전 레벨 항목과의 연결을 명시할 때는 다음 형식을 사용한다:
```markdown
<!-- 올바름: 구체적인 심화 관계 명시 -->
N3의 ～て以来（M068）가 중립적인 "~이래로"라면, ～てからというもの(は)는
"그때부터 줄곧 변화가 계속되어 왔다"는 감정적 강조와 극적인 전후 대비를 더한 표현이다.

<!-- 금지: 막연한 언급 -->
N3에서 배운 표현의 심화이다.
N4에서 학습한 내용에서 확장된다.
```

## 6. 항목 구조 및 섹션 규칙

### 6.1 항목 템플릿 구조

각 문법 항목은 다음 순서로 구성한다:
```
### ～표현（한국어 의미） ［레지스터 아이콘］
#### ■ 의미와 기능
#### ■ 접속 형태
#### ■ 예문
#### ■ 비교          ← compare_with가 있을 때만 포함
#### ■ 한국어 화자 주의  ← L1경고가 있을 때만 포함
```

섹션을 생략할 때는 섹션 자체를 쓰지 않는다（빈 섹션 헤더를 남기지 않는다）.

### 6.2 번역 위치 규칙

예문 번역은 **예문 바로 아래 한 줄**에 들여쓰기하여 배치한다:
```markdown
１. <ruby>例文<rt>れいぶん</rt></ruby>
　→ 번역
```

번역을 예문으로부터 분리하여 별도 목록으로 모으지 않는다.

### 6.3 비교박스 중복 처리 규칙

같은 비교 쌍은 **첫 등장 항목에서만 상세 비교박스를 작성**한다.  
이후 같은 챕터 내에서 다시 등장하면:
```markdown
→ ～표현명의 비교 참조
```

다른 챕터나 다른 파일(상편→하편 등)에서 재등장하면:
```markdown
→ ～표현명（Ch.X）의 비교 참조
→ ～표현명（상편 Ch.X）의 비교 참조
```

### 6.4 챕터 말미 필수 요소 순서

챕터 말미는 다음 순서로 구성한다:
```
1. 📖 장면으로 익히기
2. 📋 관련 어휘 보충   ← 해당 챕터에 어휘 항목이 있을 때만
3. ✅ 확인 체크리스트
```

### 6.5 챕터 도입부

챕터 도입부（챕터 제목과 첫 항목 사이의 설명문）는 작성하지 않는다.  
단, 챕터 전체 항목의 공통 접속 규칙이 있을 경우 **한 문장**으로 기재할 수 있다:
```markdown
<!-- 허용: 한 줄 공통 접속 안내 -->
이 챕터의 전 항목은 동사 ます형 어간에 접속한다.

<!-- 금지: 챕터 개요나 학습 목표 서술 -->
이 챕터에서는 ... 를 배운다. 학습 후에는 ...
```

### 6.6 금지 섹션

다음 섹션은 어떤 레벨에서도 포함하지 않는다:

- 색인 섹션
- 리마인더 박스
- 메타 학습 안내（"이 챕터를 마치고 나서" 등）
- `<details>` 접이식 섹션

## 7. 장면 연속 예문 규칙

### 7.1 분량 기준

| 레벨 | 최소 줄 수 | 최대 줄 수 |
|------|---------|---------|
| N5 | 6줄 | 8줄 |
| N4 | 8줄 | 12줄 |
| N3 | 10줄 | 14줄 |
| N2 | 12줄 | 16줄 |
| N1 | 14줄 | 18줄 |

"줄 수"는 A·B 등 인물의 발화 줄을 기준으로 한다（번역 줄은 제외).

### 7.2 내용 요건

- 해당 챕터의 핵심 문법 항목을 가능한 많이, **최소 3개 이상** 자연스럽게 포함한다.
- 동일 문법 항목을 장면 내에서 반복 사용하지 않는다.
- 동일 인물·소재가 **연속 3회 이상** 등장하지 않도록 예문 소재를 배분한다.
- 대화문의 경우 화자별 발화 단위로, 줄글의 경우 문장 단위로 한국어 해석을 작성한다.
- 발화자는 같은데 구어와 문어가 뒤섞이거나, 예문 혹은 한국어 해석 문장의 표현이 부자연스러울 경우 수정한다.

### 7.3 레지스터별 허용 장면 형식

| 챕터 레지스터 | 허용 장면 형식 |
|------------|------------|
| 주로 📝·📖 | 뉴스 보도문, 논설문, 소설 내러티브, 격식 서한, 공문서 형식 지문 |
| 주로 🔄·💬 | 일상 대화, 비즈니스 회화, 구어·문어 혼합 |
| 혼합 | 구어 대화 + 문어 지문 병용 가능（단, 같은 대화 내에서 레지스터 일관성 유지） |

## 8. 어휘 보충 항목 규칙

### 8.1 일반 챕터 어휘 보충 형식
```markdown
#### 📋 관련 어휘 보충

| 단어 | 의미 | 예문 |
|------|------|------|
| 단어（よみ） | 의미 | 예문。/ 번역。 |
```

### 8.2 부록 어휘 항목 형식

부록에 배치된 순수 어휘 항목은 일반 항목 템플릿 대신 다음 간략 형식을 사용한다:
```markdown
### 단어（한국어 의미） ［레지스터 아이콘］

[1~2문장 간략 설명]

| 예문 | 번역 |
|------|------|
| 예문 | 번역 |

[비교박스가 있으면 간략 형식으로]
```

## 9. JSON 데이터셋 공통 필드 규칙

### 9.1 필수 필드

모든 문법 항목 JSON 레코드는 다음 필드를 포함한다:

| 필드명 | 타입 | 설명 |
|-------|------|------|
| `id` | string | 항목 고유 식별자（레벨 접두어 포함. 예: `E001`, `M001`, `P1M001`） |
| `level` | string | JLPT 레벨（"N5", "N4", "N3", "N2", "N1"） |
| `base_form` | string | 기본 표기（예: `～について`） |
| `meaning_ko` | string | 한국어 의미 |
| `unit_type` | string | 항목 유형（"pattern", "family", "particle", "adverb" 등） |
| `register` | string | 레지스터（"spoken", "written", "formal", "literary", "mixed" 등） |

### 9.2 선택 필드

| 필드명 | 타입 | 설명 |
|-------|------|------|

| `notes` | string | 집필 시 특이 사항 |

### 9.3 ID 체계 규칙

## 10. Claude Code 편집 원칙

### 10.1 기본 원칙: 최소 개입

기존 파일을 편집할 때는 다음 원칙을 따른다:

1. **처음부터 새로 쓰지 않는다.** 기존 문서를 기준으로 편집·교정·보강한다.
2. **기존 구조를 변경하지 않는다.** 챕터 순서, 항목 순서, 섹션 구성을 임의로 바꾸지 않는다.
3. **이유가 분명한 부분만 수정한다.** 판단이 서지 않으면 수정하지 않는다.
4. **기존에 없는 항목을 임의로 추가하지 않는다.**

### 10.2 수정 우선순위

수정이 필요한 경우 다음 순서로 처리한다:

| 우선순위 | 유형 | 처리 방법 |
|---------|------|---------|
| 1순위 | 기술적 오류 | ruby 태그 쌍 오류, 닫는 태그 누락 |
| 2순위 | 언어 혼재 | 한국어 설명 문장 안에 일본어 삽입 |
| 3순위 | 규칙 위반 | ❌/⭕️ 오용, 장면 설명 일본어, 체크리스트 언어 혼재 |
| 4순위 | 레지스터 불일치 | 구어 대화에 문어 접속사, 이에 따른 ⚠️ 미처리 |
| 5순위 | 분량 미달 | 장면 연속 예문이 레벨별 최소 기준 미달 |
| 6순위 | 문체 통일 | 용어·표기 일관성 |
| 7순위 | 설명/예문 문장의 부자연스러움 | 수정 혹은 필요할 경우 문장(장면) 재작성 |

### 10.3 수정 출력 형식

편집 작업 완료 후 다음 형식으로 출력한다:
```
## 수정 요약
- [항목명 또는 위치]: [수정 내용] — [수정 이유 1줄]
...

## 수정된 최종본
（마크다운 전문）

## 미수정 항목（선택）
- [항목]: [수정하지 않은 이유]
```

### 10.4 판단 기준이 모호한 경우

다음 경우에는 수정하지 않고 주석으로 남긴다:

- 문법적으로 가능하나 자연스러움이 불확실한 예문
- 레지스터 판단이 경계에 있는 표현
- 나선형 연결 서술이 충분한지 판단하기 어려운 경우

주석 형식:
```markdown
<!-- REVIEW: [판단이 필요한 이유] -->
```

### 10.5 절대 금지 행동

편집 중 다음 행동은 어떤 이유로도 수행하지 않는다:

- 기존 항목의 챕터 배치 변경
- 집필 메모에 없는 새 항목 추가
- 비교박스 구조를 임의로 통합하거나 분리
- `<details>` 접이식 섹션 추가
- 색인, 리마인더 박스, 메타 학습 안내 추가
- 체크리스트에 문법 요소가 아닌 불필요한 일본어 추가
- 장면 설명을 일본어로 변경
