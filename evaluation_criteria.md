# JLPT 문법 교재 검수 평가 기준 (evaluation_criteria.md)

본 문서는 N5~N1 각 권 마크다운 교재(상/하편 또는 상/중/하편)에 대한 검수의 일관된 기준을
제시한다. 각 보고서는 아래 항목별 체크리스트와 발견 사항을 포함하여 작성한다.

---

## 1. 교재의 공통 구조 요약

검수 시 아래의 구조 규약을 기준으로 일관성을 평가한다.

### 1.1 파일·폴더 구성
- 레벨별 폴더(N5~N1) 안에 상/하 또는 상/중/하편 마크다운이 위치한다.
- 각 파일은 챕터(Ch.N) 단위로 구성되며, 마지막에 「부록」이 있을 수 있다.

### 1.2 문서 계층(헤딩 규약)
| 수준 | 용도 | 예시 |
|------|------|------|
| `#`  | 교재 전체 표제 | `# N5 일본어 문법 교재 — 문장의 기초를 세우다` |
| `##` | `목차` / 챕터 헤더 / `부록` | `## Ch.1 이것은 무엇인가: 기본 서술과 의문` |
| `###` | 개별 문형(또는 어휘) 항목 | `### ～だ / です（~이다） ［🔄 구어·문어 모두］` |
| `####` | 항목 내부 섹션 / 챕터 부속 블록 | `#### ■ 의미와 기능`, `#### 📖 장면으로 익히기` |

### 1.3 문형 항목의 표준 섹션 순서
1. `#### ■ 의미와 기능` — 의미·기능 설명, 한국어 의미와 사용 맥락
2. `#### ■ 접속 형태` — 품사별 접속 표(필수)
3. `#### ■ 예문` — 번호 매김 일본어 예문 + `→` 한국어 번역, 필요 시 `❌/⭕️` 오용·정용 대조
4. `#### ■ 비교` — 유사 문형 비교 표 또는 `→ ～X의 비교 참조` 상호참조
5. `#### ■ 한국어 화자 주의` (선택) — 한국어 모어 화자가 혼동하기 쉬운 점

### 1.4 챕터 부속 블록(챕터 말미)
- `#### 📖 장면으로 익히기` — 해당 챕터 문형을 종합한 짧은 대화/내러티브
- `#### 📋 관련 어휘 보충` (선택) — 보충 어휘 표
- `#### ✅ 확인 체크리스트` — 챕터 학습 목표 점검 리스트
- 챕터 첫머리에 `💡 **학습 안내**:` 블록이 올 수 있다(다항목 챕터의 학습 동선 안내).

### 1.5 레지스터(문체) 태그 — `### 항목명` 끝에 대괄호로 표기
- `［🔄 구어·문어 모두］`
- `［💬 구어］`
- `［📝 문어·격식］`
- `［🎩 경어·정중］`

### 1.6 표기 관습
- 한자에는 `<ruby>漢字<rt>かな</rt></ruby>` 형식의 후리가나를 단다.
- 일본어 예문은 「１.」「２.」… 전각 숫자, 한국어 번역은 「　→」(전각 공백+화살표)로 시작.
- 강조 기호: `⚠️`(주의·제약), `💡`(팁·참고·한국어 비교), `※`(보충 설명),
  `❌/⭕️`(오용/정용), `📖/📋/✅/💡/🔄/💬/📝/🎩`(섹션·태그 마커).
- 항목 사이 구분은 빈 줄 + `---` + 빈 줄.
- 표는 GFM 파이프 표 사용, 헤더 다음 줄에 `|---|---|` 정렬 행.

### 1.7 레벨 간 연계
- N4 이상에서는 「N5에서 배운 ～」「N3에서 배운 ～」와 같이 하위 레벨 항목을 명시적으로 참조한다.
- 동일 챕터 내 비교 표는 첫 항목에 두고, 이후 항목은 `→ ～X의 비교 참조`로 가리킨다.

## 2. 평가 영역과 항목

각 영역은 5점 척도(아래 4.항)로 평가하고, 발견 사항은 사례(파일 라인 번호 포함)와 함께 기록한다.

### A. 구조·형식 일관성 (Structure & Formatting)
- A1. 헤딩 레벨이 1.2 표를 따르는가? (오타·중복·건너뜀 없음)
- A2. 표준 섹션 4종(■ 의미와 기능 / ■ 접속 형태 / ■ 예문 / ■ 비교)이 순서대로 모두 존재하는가?
- A3. 항목 제목 끝 레지스터 태그 4종 중 정확히 하나가 부착되어 있는가?
- A4. 챕터 말미 `📖 장면으로 익히기`와 `✅ 확인 체크리스트`가 존재하는가?
- A5. 항목 구분자(`---`)와 빈 줄이 일관되게 사용되는가?
- A6. 표(접속/비교)의 컬럼 수·정렬이 일관되며, 빈 칸 처리(`—`)가 명확한가?
- A7. 목차의 항목·순서가 본문의 실제 항목·순서와 일치하는가?

### B. 내용 정확성 (Linguistic Accuracy)
- B1. 의미·기능 설명이 표준 일본어 문법 기술과 부합하는가? 과도한 단순화·오해 소지 없음.
- B2. 접속 규칙이 정확한가? (1·2·3그룹 동사, い/な형용사, 명사, ない형/て형/た형 등의 활용 정확)
- B3. 활용 패러다임 표(수동·사역·가능 등)에 누락·오류 없음.
- B4. 예문의 일본어 자체가 자연스럽고 문법적으로 올바른가?
- B5. 한국어 번역이 의미를 정확히 전달하며 자연스러운가?
- B6. `❌/⭕️` 오용·정용 대조가 실제 학습자가 흔히 범하는 오류를 정확히 반영하는가?
- B7. 후리가나(`<ruby>` 태그)의 한자·읽기·태그 균형(닫힘)이 정확한가?
- B8. 레지스터 태그가 문형의 실제 사용역과 일치하는가? (예: 「ねばならない」를 구어로 표시하지 않았는지)

### C. 교수·학습 설계 (Pedagogical Design)
- C1. 항목 배열이 같은 챕터 안에서 의미·기능적으로 응집성을 가지는가?
- C2. 다항목 챕터에 `💡 학습 안내`가 적절히 제공되어 학습 동선이 잡혀 있는가?
- C3. 비교 섹션이 유사·혼동 항목 간의 차이를 분명히 드러내는가?
- C4. 「한국어 화자 주의」가 필요한 항목에 빠짐없이 들어 있는가? (한·일 대응이 어긋나는 항목 우선)
- C5. 예문이 한 항목당 최소 3개 이상이며, 다양한 문맥(일상/공적/문학 등)을 보이는가?
- C6. `📖 장면으로 익히기`가 해당 챕터의 핵심 문형들을 자연스럽게 통합하는가?
- C7. `✅ 확인 체크리스트`가 챕터의 핵심 학습 목표를 망라하는가?

### D. 레벨 적합성 (JLPT Level Appropriateness)
- D1. 수록 항목이 해당 JLPT 레벨(N5~N1)의 표준 문형 범위에 합당한가? (레벨 이탈 항목이 있는지)
- D2. 설명·예문에 사용된 어휘·한자 수준이 해당 레벨에 맞는가? (지나치게 어려운 한자/어휘 남용 없음)
- D3. 한 항목의 분량과 깊이가 레벨에 비례하는가? (N5는 간결, N1은 수사적 설명까지)
- D4. 상편→하편(또는 상→중→하)으로 갈수록 난이도가 자연스럽게 상승하는가?

### E. 레벨 간·항목 간 연계 (Cross-Reference & Progression)
- E1. 상위 레벨 항목이 하위 레벨 관련 항목을 명시적으로 인용·연결하는가? (예: 「N4에서 배운 ～なければならない에서 심화…」)
- E2. 「→ ～X의 비교 참조」 상호참조가 실제 해당 항목을 가리키며 끊김이 없는가?
- E3. 동일 형태의 동형이의(예: 「のに」 목적/역접, 「に対して」 향함/대조)가 한 챕터 안에서 명시적으로 구분되는가?
- E4. 같은 챕터 내 첫 항목의 비교 표가 후속 항목과 모순되지 않는가?

### F. 표기·체재 (Surface & Typography)
- F1. 후리가나 태그 (`<ruby>` `<rt>`) 의 열고 닫음이 모두 균형(unbalanced 없음)이며, 한자에 빠짐없이 부여되어 있는가?
- F2. 일본어/한국어/숫자 사이의 공백, 전각·반각 사용이 일관되는가?
   - 일본어 예문 번호: 전각 「１.」「２.」, 번역 줄 시작 「　→」
- F3. 마크다운 표가 렌더링 시 깨지지 않는 형식인가? (셀 수 일치, 헤더 분리줄 존재)
- F4. 마커 이모지(🔄/💬/📝/🎩/⚠️/💡/📖/📋/✅/❌/⭕️)가 1.5·1.6의 용도와 일치하게 사용되는가?
- F5. 띄어쓰기·문장부호(쉼표 「、」「,」「，」, 마침표 「。」「.」) 사용이 한 파일 내에서 일관되는가?
- F6. 「( )」와 「（ ）」(반각/전각 괄호) 사용이 일관되는가?

### G. 콘텐츠 누락·중복 (Coverage)
- G1. 목차에는 있는데 본문에 누락된 항목이 있는가? 또는 그 반대.
- G2. 한 챕터 안에서 중복 등재된 항목이 있는가?
- G3. 중요한 표준 문형(JLPT 출제 빈도 높은)이 빠져 있지는 않은가?
- G4. 비교 표에서 언급된 항목이 본문에 실제 등재되어 있는가?

## 3. 보고서 작성 형식

`reports/{filename}_report.md` 파일은 다음 섹션을 포함한다.

```
# {파일명} 검수 보고서

## 0. 개요
- 파일: <경로>
- 라인 수 / 챕터 수 / 항목 수:
- 검수일: YYYY-MM-DD
- 종합 평점: ★★★★☆ (4.2/5)

## 1. 영역별 평가
| 영역 | 점수(1~5) | 핵심 코멘트 |
| A. 구조·형식 일관성 | | |
| B. 내용 정확성 | | |
| C. 교수·학습 설계 | | |
| D. 레벨 적합성 | | |
| E. 레벨 간·항목 간 연계 | | |
| F. 표기·체재 | | |
| G. 콘텐츠 누락·중복 | | |

## 2. 발견 사항(Findings)
- [심각도] [영역 코드] (라인) 문제 요약 — 권장 수정안
  예) [Major][B2] (L.412) 「死ぬ」를 1그룹으로 분류하지 않고 표에서 누락 — 「死なれる」 행 추가 권장.

## 3. 우수 사례
- 잘 설계된 비교 표/장면/체크리스트 등 모범 사례 정리.

## 4. 종합 의견 및 수정 권고 우선순위
1) 즉시 수정 필요(Major/Critical)
2) 가독성·일관성 개선(Minor)
3) 추가 보강 제안(Suggestion)
```

### 3.1 발견 사항 분류 표기
- 심각도: `Critical` / `Major` / `Minor` / `Suggestion`
- 영역 코드: A1~G4 중 해당 항목 코드를 함께 표기
- 위치: 가능한 한 라인 번호(`L.123`) 또는 항목명(`### ～たほうがいい`)을 명시

### 3.2 점수 척도(영역별 1~5)
- 5: 결함 없음, 모범적
- 4: 사소한 결함만 존재
- 3: 일부 항목에서 일관성·정확성 결함, 사용에는 큰 문제 없음
- 2: 다수 결함, 학습자 혼란 우려
- 1: 광범위한 오류·결손, 사용 곤란

## 4. 검수 절차 권장 순서

1. 파일 전체 라인 수·챕터 수 확인 후 목차와 본문의 일대일 대응을 먼저 점검(영역 A·G).
2. 챕터별로 항목을 순회하며 표준 4섹션 존재 여부와 레지스터 태그 확인(영역 A).
3. 각 문형의 의미·접속·예문·비교 정확성을 점검(영역 B).
4. 챕터 말미 `📖`·`✅` 블록 존재·완성도 점검(영역 C).
5. 동일 챕터·인접 챕터의 비교 표와 상호참조 정합성 점검(영역 E).
6. 후리가나 태그 균형, 마크다운 표 형식, 마커 사용 일관성 등 표기 검사(영역 F).
7. 해당 레벨에 비해 과/부족한 항목 검토(영역 D·G).
8. 보고서를 `reports/{filename}_report.md`로 저장.

## 5. 자동 점검에 활용 가능한 패턴(참고)

- 헤딩 패턴: `^### ～.+［(🔄|💬|📝|🎩) .+］$` (레지스터 태그 누락 탐지)
- 표준 섹션: 한 항목 안에 `#### ■ 의미와 기능`, `#### ■ 접속 형태`, `#### ■ 예문`, `#### ■ 비교`가 모두 등장하는지.
- 후리가나 균형: `<ruby>` 개수 == `</ruby>` 개수 == `<rt>` 개수 == `</rt>` 개수.
- 챕터 말미: `## Ch.` 사이에 `📖 장면으로 익히기`와 `✅ 확인 체크리스트`가 1회 이상 등장.
- 예문 번호: 일본어 예문이 전각 숫자 「１.」 형태로 번호 매김 되어 있는지.
- `→` 번역 줄: `　→`(전각 공백 + 화살표)로 시작하는지.
